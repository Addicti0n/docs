Lava uses the Cosmos SDK bank & account modules for seamless integration with existing Cosmos wallets.

After setting up a wallet, it is highly recommended to test using it. You can test it using the [faucet](/faucet#discord-faucet).
Get testnet tokens from the faucet, and execute a transaction by sending them to another account address and then back.

## Choose a wallet

Popular wallet options include:
* [Keplr](#keplr)
* [Leap](#leap)
* Advanced: [CLI wallet](#cli)
* Advanced: [Other wallets](#other)

## CLI wallet {#cli}

:::warning experimental 
This section is for advanced users only
:::

The command line (CLI) wallet offers multiple backends including a hardware wallet.

## Prepare an account using the CLI {#account}
Set your preferred keyring backend ([FAQ: what is a keyring](/faq#keyring)).

```bash
lavad config keyring-backend [os|file|kwallet|pass|test|memory]
```


### Add a key
:::caution
Read and understand the of saving your private key in your chosen keyring
:::

1. Replace $ACCOUNT_NAME with your chosen account name:

```bash
current_lavad_binary="$HOME/.lava/cosmovisor/current/bin/lavad"
ACCOUNT_NAME="name_here"
$current_lavad_binary keys add $ACCOUNT_NAME
```

:::danger
Write down your mnemonic in a safe place, it can not recovered. 
:::

2. Show and lookup your account in your keys list:

```bash
$current_lavad_binary keys show $ACCOUNT_NAME
$current_lavad_binary keys list
```

:::caution Pencils out üìù
Your public address starts with `lava@`
:::

#### Recovering an account

1. To recover an account using your existing mnemonics:
```bash
lavad keys add $NEW_NAME --recover
```
You will be prompted to enter your seed phrase.

<br/>

2. Check your account was successfully imported:

```bash
$current_lavad_binary keys show $ACCOUNT_NAME
```

### Hardware wallet with the CLI
:::info
These instructions are for the Ledger hardware wallet.
Your device should be up-to-date and with the Cosmos application installed.
:::

Register your ledger wallet with $ACCOUNT_NAME:
```bash
$current_lavad_binary keys add $ACCOUNT_NAME --ledger
```


:::info
Always add the `--ledger` flag to transactions commands (lavad tx ...) when using Ledger
:::

```bash
$current_lavad_binary tx bank send $ACCOUNT_NAME <lava@target-account> 10000ulava --from $ACCOUNT_NAME --ledger
```

For more information read https://github.com/cosmos/gaia/blob/main/docs/resources/ledger.md (Use `$current_lavad_binary` for the command).

## Multisig

### Creating a multisig

1. Import the public keys of the signers:
```bash
$current_lavad_binary keys add multisig_imported_alice --pubkey '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data1..."}'
$current_lavad_binary keys add multisig_imported_bob --pubkey '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data2..."}'
```

2. Lookup the imported keys:
```bash
$current_lavad_binary keys show wallet_imported
- address: lava@xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
  name: wallet_imported
  pubkey: '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data..."}'
  type: offline

$current_lavad_binary keys show wallet_generated_keyring
- address: lava@xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
  name: wallet_generated_keyring
  pubkey: '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data..."}'
  type: local
```

3. Create an account with the imported keys:
```bash
$current_lavad_binary keys add $MULTISIG_ACCOUNT_NAME --multisig multisig_imported_alice,multisig_imported_bob --multisig-threshold $THRESHOLD
```

* **$THRESHOLD** - defines the number of signers required for a valid signature

4. Get the multisig public address with 
```bash
$current_lavad_binary keys show $MULTISIG_ACCOUNT_NAME
```

* Example output:
```bash
$current_lavad_binary keys add multisig2_of_3 --multisig wallet1,wallet2,wallet3 --multisig-threshold 2

- address: lava@xxxxx...
  name: multisig2_of_3
  pubkey: '{"@type":"/cosmos.crypto.multisig.LegacyAminoPubKey","threshold":2,"public_keys":[{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data1..."},{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data2..."},{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data3..."}]}'
  type: multi
```

### Signing transactions

To sign a multisig transaction, etach signer signs it separately offline.

#### Generate the transaction
Alice and bob have their private keys in their own environment, and created a multisig in a shared machine
```bash
$current_lavad_binary tx bank send $(lavad keys show alice-bob-multisig -a) lava@destination-address 10000000ulava  --generate-only --chain-id lava-testnet-2 > tx_bank.json
```

#### Signing it
Copy the transaction to signer's environment and sign it.
Use an rpc to fetch account number and sequence for you or set them manually via flags.
```bash
lavad tx sign --from $(lavad keys show -a alice-ledger) --multisig lava@multisig-address tx.json --sign-mode amino-json --chain-id lava-testnet-2 >> tx-signed-alice.json
or
lavad tx sign --from $(lavad keys show -a alice-ledger) --multisig lava@multisig-address tx.json --sign-mode amino-json --chain-id lava-testnet-2 --account-number X --sequence X >> tx-signed-alice.json
```

A second signe in a different environment:
```bash
lavad tx sign --from $(lavad keys show -a bob-ledger) --multisig lava@multisig-address tx.json --sign-mode amino-json --chain-id lava-testnet-2 >> tx-signed-bob.json
```

#### Sending it
1. Creating the signed multisig transaction
2. Broadcasting it
```bash
lavad tx multisign --from alice-bob-multisig tx.json alice-bob-multisig tx-signed-alice.json tx-signed-bob.json --chain-id lava-testnet-2 > tx_ms.json
lavad tx broadcast ms/tx_ms.json --chain-id lava-testnet-2
```

### Other wallets {#other}

:::warning experimental 
This section is for advanced users only
:::
You can use the following arguments to setup a different wallet no described in this manual.

* **coinType** - 118 (standard cosmos wallet type)
* **minimal denom** - ulava
* **prefix** - lava@
* **coinDecimals** - 6
* **rpc** - "https://public-rpc.lavanet.xyz"
* **rest** - "https://public-rpc.lavanet.xyz/rest"
