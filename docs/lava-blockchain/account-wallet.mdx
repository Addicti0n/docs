Lava uses Cosmos bank & account modules and enables you to use any standard Cosmos wallet.

## Create Your Wallet

there are several ways to create your wallet or integrate an existing one, it is possible to use existing cosmos wallets such as Keplr, Leap or the cli tools lavad, lavap

### lava wallet details
* **coinType** - 118 (standard cosmos wallet type)
* **minimal denom** - ulava
* **prefix** - lava@
* **coinDecimals** - 6

## Leap Usage {#leap}
in order to use Leap wallet, under Switch Chains menu, click Add New Chain (located at the bottom) then enter the following details:
* **Chain id** - lava-testnet-2
* **Chain Name** - Lava Testnet
* **New RPC URL** - https://public-rpc.lavanet.xyz
* **New REST URL** - https://rest-public-rpc.lavanet.xyz
* **Address Prefix** - lava@
* **Native Denom** - ulava
* **Coin Type** - 118
* **Decimals** - 6
* **Block Explorer URL** - https://lava.explorers.guru

your chain should be added, and you should see your balance within the wallet, you can now sign transactions with this wallet or use it's send functions

# using the cli {#cli}

there are several ways to use the cli, it is possible to connect to your hardware ledger device
we will describe them in the following section

## Prepare an account using the CLI {#account}
First need to add a wallet to your keyring ([FAQ: what is a keyring](faq#keyring)).

you should set your preferred keyring backend using

```bash
lavad config keyring-backend [os|file|kwallet|pass|test|memory]
```


### using the keyring to store your private key
:::caution
make sure you understand the risks of saving your private key in the keyring
:::
You may use your seed phrase to import an existing wallet, described later

replace $ACCOUNT_NAME with a name of your choosing:

```bash
current_lavad_binary="$HOME/.lava/cosmovisor/current/bin/lavad"
ACCOUNT_NAME="name_here"
$current_lavad_binary keys add $ACCOUNT_NAME
```

:::danger
Ensure you write down the mnemonic as you can not recover the wallet without it. 
:::

To ensure your wallet was saved to your keyring, use show or look for the `ACCOUNT_NAME` is in your keys list:

```bash
$current_lavad_binary keys show $ACCOUNT_NAME
$current_lavad_binary keys list
```

#### recovering an account into the keyring

if you already have an account generated in a different place and want to store it in the keyring for usage it is possible to do so using the mnemonic
```bash
lavad keys add $NEW_NAME --recover
```
you will be prompted to enter your seed phrase

<br/>
to make sure your account was successfully imported use 

```bash
$current_lavad_binary keys show $ACCOUNT_NAME
```

### using ledger in the cli
it is possible to use a hardware wallet, this will not import your private key on the machine,
:::info
make sure your ledger is updated to work with cosmos
:::

```bash
$current_lavad_binary keys add $ACCOUNT_NAME --ledger
```
the cli now knows to work with your ledger device whenever the $ACCOUNT_NAME is used

:::info
it is necessary to add the --ledger flag to transactions commands (lavad tx ...) when using ledger
:::

```bash
$current_lavad_binary tx bank send $ACCOUNT_NAME <lava@target-account> 10000ulava --from $ACCOUNT_NAME --ledger
```

further clarifications are existing in: ($current_lavad_binary is the command to use)
https://github.com/cosmos/gaia/blob/main/docs/resources/ledger.md

# multisig

## creating a multisig
it is possible to generate multisig transactions and addresses using only public addresses. you will need to import the public key for this.
```bash
$current_lavad_binary keys show wallet_imported
- address: lava@xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
  name: wallet_imported
  pubkey: '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data..."}'
  type: offline

$current_lavad_binary keys show wallet_generated_keyring
- address: lava@xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx...
  name: wallet_generated_keyring
  pubkey: '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data..."}'
  type: local
```

importing a public key is performed with
```bash
$current_lavad_binary keys add multisig_imported_alice --pubkey '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data1..."}'
$current_lavad_binary keys add multisig_imported_bob --pubkey '{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data2..."}'
```

now with the keys imported you will need to create a multi sig account
```bash
$current_lavad_binary keys add $MULTISIG_ACCOUNT_NAME --multisig multisig_imported_alice,multisig_imported_bob --multisig-threshold $THRESHOLD
```

* **$THRESHOLD** - defines how many signers are required for a valid signature

you can check your account and see it's public address with 
```bash
$current_lavad_binary keys show $MULTISIG_ACCOUNT_NAME
```

this should look like:
```bash
$current_lavad_binary keys add multisig2_of_3 --multisig wallet1,wallet2,wallet3 --multisig-threshold 2

- address: lava@xxxxx...
  name: multisig2_of_3
  pubkey: '{"@type":"/cosmos.crypto.multisig.LegacyAminoPubKey","threshold":2,"public_keys":[{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data1..."},{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data2..."},{"@type":"/cosmos.crypto.secp256k1.PubKey","key":"data3..."}]}'
  type: multi
```

## signing transactions

signing multisig transactions requires separate signing, this is done offline as it requires each signer to privately sign

### generating the transaction
alice and bob generated their private keys in their own environment, and created a multisig in a shared machine
```bash
$current_lavad_binary tx bank send $(lavad keys show alice-bob-multisig -a) lava@destination-address 10000000ulava  --generate-only --chain-id lava-testnet-2 > tx_bank.json
```

### signing it
copying the transaction to the local separate environment of the signers, then signing it. this requires either a connection to the rpc (to fetch account number and sequence for you) or setting account-number and sequence manually via flags

```bash
lavad tx sign --from $(lavad keys show -a alice-ledger) --multisig lava@multisig-address tx.json --sign-mode amino-json --chain-id lava-testnet-2 >> tx-signed-alice.json
or
lavad tx sign --from $(lavad keys show -a alice-ledger) --multisig lava@multisig-address tx.json --sign-mode amino-json --chain-id lava-testnet-2 --account-number X --sequence X >> tx-signed-alice.json
```

and a different signer on his local environment
```bash
lavad tx sign --from $(lavad keys show -a bob-ledger) --multisig lava@multisig-address tx.json --sign-mode amino-json --chain-id lava-testnet-2 >> tx-signed-bob.json
```

### creating the signed multisig tx, then broadcast it
```bash
lavad tx multisign --from alice-bob-multisig tx.json alice-bob-multisig tx-signed-alice.json tx-signed-bob.json --chain-id lava-testnet-2 > tx_ms.json
lavad tx broadcast ms/tx_ms.json --chain-id lava-testnet-2
```
